## **Sprint 3 Retrospective**

What went well:

* App UI was successfully polished (as seen in the final Home, Habits List, Daily Log, and Stats screenshots).  
* Team successfully collaborated to generate the final release APK (flutter build apk \--release) for the Advance Mobile Application Development course submission.

What could be improved:

* The repository currently shows 0 Pull Requests and 0 Issues used. We relied mostly on external communication rather than tracking bugs and code reviews directly in GitHub. 

* Minor UI inconsistencies between iOS and Android emulators were caught late in the testing phase.

Action items for next sprint:

* Utilize GitHub Issues and Pull Requests more actively to document QA findings and code reviews.  
* Conduct cross-platform UI testing earlier in the sprint to catch styling bugs.
